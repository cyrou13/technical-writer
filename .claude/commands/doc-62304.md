Exécute le pipeline complet de génération documentaire IEC 62304 Classe A.
Sortie locale en Markdown dans `docs/`. Ne pousse rien à l'extérieur.

## Étapes

1. **Cartographier** le repo en lançant le sub-agent `code-archeologist`.
   Bloquant — attendre la fin avant l'étape 2.

2. **Rédiger SRS en premier** : `requirements-writer` → `docs/items/SRS/`.
   Bloquant pour que les agents suivants disposent des IDs SRS.

3. **Rédiger en parallèle** (un seul message, plusieurs Agent calls) :
   - `architecture-writer` → `docs/items/SDS/` (remplit `links.implements`).
   - `test-evidence-collector` → `docs/items/TC/` (remplit `links.verifies`).

4. **Analyse de risques** : lancer `risk-analyst`. Il écrit dans
   `docs/items/RSK/`, peut créer de nouvelles SRS de mitigation
   (`priority: Must`, `links.mitigates`), et peut ajouter
   `links.mitigates` à des items SDS/TC existants. Bloquant.

5. **Build** : `python tools/build_docs.py`. Vérifier que tous les
   fichiers `docs/generated/{10_SRS,20_SDS,30_test_evidence,40_traceability,50_risk_analysis,_to_implement}.md`
   et `coverage.json` ont été produits.

6. **Revue** : `compliance-reviewer` écrit
   `docs/generated/99_compliance_review.md`.

7. **Synthèse à l'utilisateur** (≤ 12 lignes) :
   - métriques de couverture (impl + vérif) depuis `coverage.json`,
   - nombre de risques ouverts / RSK avec résiduel non acceptable
     (alerte si > 0),
   - nombre d'items dans `_to_implement.md` (backlog actionnable),
   - nombre de gaps bloquants vs non-bloquants,
   - chemins des principaux livrables.

## Garde-fous

- Si `_codemap.md` n'est pas produit à l'étape 1 → arrêter et demander à
  l'utilisateur.
- Si le build échoue → afficher l'erreur Python, ne pas masquer.
- Ne jamais commit/push sans demande explicite — toute la sortie est
  locale.
- Argument optionnel `$ARGUMENTS` : si non vide, le passer comme
  périmètre aux agents (ex. `/doc-62304 packages/auth` pour limiter le
  scope au package auth).
